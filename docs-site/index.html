topicmd

A docs-as-code toolchain for semantic structure, reusable content, navigation profiles, i18n coverage, and AI-assisted topic management on Markdown projects. MDX is opt-in.

Philosophy Architecture CLI Schema VS Code

Core philosophy

Documentation is not "code with text files" — it is relationships with files. The file system provides physical storage. Semantic structure, relationships, navigation, audience targeting, and language variants are separate concerns that should be explicitly modelled, not inferred from folder layout.

Architecture

topicmd is organised in decoupled layers:

Content layer
Topics (.md, .mdx opt-in) are navigable, indexed units. Fragments (fragments/) are reusable snippets included via <!-- @include: ../fragments/x.md --> — never navigable. Variables come from docs.vars.yaml and are referenced as {{vars.*}}. Locale variants use a filename suffix: topic.de.md.
Schema layer — docs.schema.yaml
Defines the content model: configurable dimensions (filter axes), profiles (named partial filter combinations), topic types (required/optional fields and structural contracts), and i18n configuration. Core hardcodes no dimension or profile names.
Navigation layer — nav/*.yaml
Manifests reference topics by path; topics never know their nav position. topicmd generates a filtered manifest per profile plus a profiles.json for optional client-side switching — static and framework-agnostic.
Index layer — docs.index.json
The generated, primary tooling/AI-context artifact (git-ignored). It records topics, fragments, relationships (links, orphans, suspected duplicates, fragment usage), coverage per profile and gaps, validation diagnostics, i18n coverage, and stale translations.

CLI commands

Command Purpose
topicmd validate Validate all topics against the schema; exit 1 on errors.
topicmd index Generate docs.index.json.
topicmd scaffold <type> Create a new topic with valid frontmatter.
topicmd nav build Generate base + per-profile nav manifests and profiles.json.
topicmd i18n status Report i18n coverage and stale translations.

All commands exit non-zero on failure, so they slot into CI without extra configuration.

Schema example

dimensions:
  - id: product
    label: Product
    values: [alpha, beta]
  - id: audience
    label: Role
    values: [developer, admin]

profiles:
  - id: dev-beginner
    label: "Developer – Getting Started"
    filters:
      product: alpha
      audience: developer

topic_types:
  task:
    required: [title, topic_type]
    contract:
      must_contain: steps

i18n:
  default: en
  locales: [de, fr]
  strategy: suffix

VS Code extension

  • Topic Health panel — orphans, missing fields, coverage gaps, and stale translations from the index.
  • Frontmatter intelligence — completion for topic_type and dimension values, plus inline diagnostics.
  • Quick Scaffold — create a new topic from a topic type with valid frontmatter.
